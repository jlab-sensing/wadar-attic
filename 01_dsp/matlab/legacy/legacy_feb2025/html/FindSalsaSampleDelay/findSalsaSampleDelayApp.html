Description

The radar register setting SampleDelayToReference is a timing variable to indicate to the radar receiver when to start listening with respect to the transmitted pulse. Finding the correct Sample DelayToReference delays the radar receiver acquisition to in time and distance. The idea is to artificially move the "zero" refence plane to the desired impingement point of the antennas so absolute ranging is accurate. Essentially, this App helps the user calculate and compensate for the timing delays in the chip, PCB traces, cables, and antennas to move the "zero" plane to the antenna phase center.

Setup

  1. Setup the Beaglebone Black (BBB) and Salsa cape with the desired antennas and cables for future use. Note: max cable length, approximately 5 meters (Contact FlatEarth if desired cable length is beyond 5 meters).
  2. Find a suitable target with the following specifications: the target size or target radar cross-section (RCS) needs to be equivalent to or bigger than 650 sq. cm. (~100 sq. inches) metal plate. Note: a 12 inch x 9 inch copper-clad sheet of FR4 works well.
  3. Separate the unit from the target and point the antennas at the target either by pointing it at a wall with a hanging target or proping the target above the radar using a relatively radar transparent structure, e.g. an elongated cardboard box works well.Note: Target must physically be within 1 radar frame from antenna (i.e. 1 meter for Ancho and 2 meters for Cayenne).

Connection

Connect to the radar using the default USB virtual port, 192.168.7.2, or specify an IP address to use via Ethernet (Insert Cat5e cable into RJ-45 jack on the BBB and reboot so DHCP server assigns the BBB a unique IP address) Easily open or close the radar connection with the labeled push buttons.

The radar always performs a calibration immediately after connection to accurately calculate all the internal delays for accurate SampleDelayToReference calibartion. The App alerts the user when calibration is finished and enables all the remaining operation buttons. Note: Calibration time for the Ancho is ~13 secs and ~2 secs for the Cayenne.

Operation & Output

First, the proper Cape type must be choosen as some radar register settings are unique to the cape type. Choose either Ancho or Cayenne; only one can be choosen at a time.

Next, measure the true target distance to the desired "zero" antenna plane, to the nearest half-centimeter. Target must physically be within 1 radar frame from antenna (i.e. 1 meter for Ancho and 2 meters for Cayenne).

Once the true target distance, in meters, is entered hit the "Run" push button. There are two options after this:

  1. Use a background sky shot for clutter removal, i.e. more robust measurements, but takes more time as the target needs to be removed for data capture, then put back in place.
  2. Do NOT use a background sky shot (this will be used for speed, but can still deliver accurate timing measurements).

If YES, first remove the target for Background clutter capture, and these two dialog boxes are encountered, hit "YES" and proceed; otherwise, the App begins.

Once the App is running it is toggling through different delays, capturing radar measurments, and calculating target position. The user will see screens similar to this:

When data capture is finished, a linear regression is begun on the data points to find the most accurate SampleDelayToReference. The measurement is checked in two ways: if it has an acceptable Quality Index and if it is within a certain tolerance from the Direct Pulse. If it passes both, the App accepts and presents the results

Otherwise, adjust your setup and try again!
